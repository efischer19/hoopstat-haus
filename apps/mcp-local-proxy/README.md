# Hoopstat MCP Local Proxy

A lightweight MCP (Model Context Protocol) proxy adapter that runs locally and
provides AI agents with access to Hoopstat Haus NBA statistics data. It also
doubles as a human-readable command-line tool for querying the same data
directly from your terminal.

## Overview

This adapter translates MCP JSON-RPC queries from AI agents into standard HTTP
GET requests targeting static JSON artifacts served via CloudFront. All compute
runs locally -- the cloud serves only static data.

## Installation

```bash
# Via uvx (recommended)
uvx hoopstat-mcp

# Via pip
pip install hoopstat-mcp-proxy
```

## Configuration

Set the `HOOPSTAT_BASE_URL` environment variable to override the default
CloudFront endpoint:

```bash
export HOOPSTAT_BASE_URL="https://data.hoopstat.haus"
```

## Usage

### CLI Mode (default)

Run any command directly for human-readable, pretty-printed JSON output:

```bash
# Show available commands
hoopstat-mcp --help

# Fetch the latest data index
hoopstat-mcp get-index

# Fetch a specific player's daily stats
hoopstat-mcp get-artifact player_daily/2024-11-15/2544

# Fetch a team's daily stats
hoopstat-mcp get-artifact team_daily/2024-11-15/1610612747

# Fetch a top-scorers leaderboard
hoopstat-mcp get-artifact top_lists/2024-11-15/points
```

### MCP Server Mode

Pass the `--mcp` flag to start the JSON-RPC server over stdio for AI clients:

```bash
hoopstat-mcp --mcp
```

### Claude Desktop Configuration

Add to your Claude Desktop MCP config:

```json
{
  "mcpServers": {
    "hoopstat": {
      "command": "uvx",
      "args": ["hoopstat-mcp", "--mcp"]
    }
  }
}
```

## Available Tools

### `get_index`

Fetches the latest data index listing all available datasets, dates, and
resource URIs.

### `get_artifact`

Fetches a specific JSON artifact by resource URI. Examples:

- `player_daily/2024-11-15/2544` -- LeBron James' stats for a given date
- `team_daily/2024-11-15/1610612747` -- Lakers' team stats
- `top_lists/2024-11-15/points` -- Top scorers list

## Default Resources Exposed

By default, the proxy exposes the same public JSON artifacts generated by the
Gold analytics layer and served through CloudFront.

### Index

- `index/latest.json` -- the latest published index document containing:
  - the most recent available data date
  - links or resource URIs for the default public datasets

### Player Daily Artifacts

- `player_daily/{date}/{player_id}.json`
- Example: `player_daily/2024-11-15/2544`

These contain one player-level daily stats artifact per player per date.

### Team Daily Artifacts

- `team_daily/{date}/{team_id}.json`
- Example: `team_daily/2024-11-15/1610612747`

These contain one team-level daily stats artifact per team per date.

### Top Lists

- `top_lists/{date}/points.json`
- `top_lists/{date}/efficiency_rating.json`
- `top_lists/{date}/true_shooting_percentage.json`
- `top_lists/{date}/assists.json`
- `top_lists/{date}/rebounds.json`

These contain leaderboard-style artifacts for the latest published date.

## Resource URI Notes

- `get_artifact` accepts a resource URI with or without the `.json` suffix
- the recommended way to discover valid URIs is to call `get_index` first
- if a resource does not exist, the proxy returns a clean MCP error message

## Development

```bash
cd apps/mcp-local-proxy
poetry install
poetry run pytest
poetry run ruff format .
poetry run ruff check .

# Run the CLI locally
poetry run hoopstat-mcp get-index

# Run the MCP server locally
poetry run hoopstat-mcp --mcp

# Use MCP inspector
poetry run mcp dev app/server.py

# Override data endpoint
HOOPSTAT_BASE_URL="http://localhost:8000" poetry run hoopstat-mcp get-index
```
